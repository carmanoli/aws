AWS Lab 18 – Amazon EventBridge Scheduler with AWS Lambda

Objective

Learn how to invoke an AWS Lambda function automatically using Amazon EventBridge Scheduler.

This lab introduces scheduled serverless execution while reusing the existing infrastructure built in previous labs.

Step 1 - Open Amazon EventBridge Scheduler

  1. Open the AWS Console.
  2. Search for:
    3. EventBridge Scheduler
  3. Open the service.
  4. Select:
    5. Create schedule

Step 2 - Configure Schedule Details

Schedule Name

lab-lambda-schedule

Description

Invoke lab Lambda every 5 minutes

Schedule Group

default

Configure:
Setting Value

Time Zone Europe/Lisbon

Flexible Time Window Off
Leave the optional Start Date and End Date empty.

Step 5 - Configure Settings

Leave the default values:
Setting Value
Schedule State Enable
Retry Policy Disabled
Dead-letter Queue None
Encryption AWS Managed Key
Execution Role Create new role for this schedule

Step 6 - Review and Create

Review the configuration and select: 
Create schedule
AWS automatically creates an IAM execution role allowing the Scheduler to invoke the Lambda function.

Troubleshooting

Lambda Function Does Not Appear

Verify that you are using the correct AWS Region. This lab uses: 
eu-north-1 (Europe - Stockholm)
If another region (such as us-east-1) is selected, the Lambda function will not appear.

No Log Entries

Verify:
  • The schedule is enabled.
  • The Lambda function exists.
  • The correct region is selected.
  • Wait at least five minutes after creating the schedule.

Key Takeaways

  • EventBridge Scheduler executes workloads at predefined intervals.
  • The Scheduler uses an IAM execution role to invoke AWS services securely.
  • A Lambda function can be triggered by multiple AWS services without changing its code.
  • Scheduler-based automation is commonly used for maintenance tasks, reports, backups, monitoring, and scheduled serverless workloads.
