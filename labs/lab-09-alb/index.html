Lab 09

AWS Application Load Balancer

Create an internet-facing Application Load Balancer and distribute HTTP traffic across two Amazon Linux 2023 EC2 instances.

EC2ALBTarget GroupsHealth Checks

Objectives

  • Install Apache on two EC2 instances.
  • Create a target group and register both instances.
  • Create an internet-facing Application Load Balancer.
  • Test load distribution, health checks and automatic failover.

Architecture

Internet
   │
   ▼
Application Load Balancer (HTTP :80)
   │
   ▼
Target Group
   ├── EC2-1 · Apache
   └── EC2-2 · Apache

Prerequisites

  • Default VPC
  • Two Amazon Linux 2023 EC2 instances
  • An IAM role with AmazonSSMManagedInstanceCore
  • Working Session Manager access

1. Install and start Apache

Run on each EC2 instance:

sudo dnf install -y httpd
sudo systemctl enable --now httpd
sudo systemctl status httpd

2. Create different pages

EC2-1

echo "<h1>Server 1 - EC2 behind the ALB</h1>" | sudo tee /var/www/html/index.html

EC2-2

echo "<h1>Server 2 - EC2 behind the ALB</h1>" | sudo tee /var/www/html/index.html

Test locally with curl localhost.

3. Create the target group

Open EC2 → Target Groups → Create target group and configure:

SettingValue
Target typeInstances
Nametg-web-lab9
Protocol / portHTTP / 80
VPCThe EC2 instances' VPC
Health check path/

Register both instances. They may initially appear as Unused until the target group is connected to a load balancer.

4. Configure security groups

ALB security group: SG-ALB

TypePortSource
HTTP800.0.0.0/0

Keep the default outbound rule allowing all traffic.

EC2 security group

TypePortSource
HTTP80SG-ALB
Referencing the ALB security group as the source means that the web servers do not need to accept HTTP directly from the internet.

5. Create the Application Load Balancer

Open EC2 → Load Balancers → Create load balancer:

  • Type: Application Load Balancer
  • Scheme: Internet-facing
  • IP address type: IPv4
  • Listener: HTTP on port 80
  • VPC: the same VPC as the EC2 instances
  • Subnets: select two Availability Zones
  • Security group: SG-ALB
  • Default action: forward to tg-web-lab9

6. Verify health checks

Open EC2 → Target Groups → Targets. Possible states include Initial, Healthy, Unhealthy and Unused. When both targets are healthy, the ALB can distribute traffic.

7. Test load balancing

Open http://<ALB-DNS-name> and refresh several times. The expected responses alternate between Server 1 and Server 2.

8. Simulate a failure

Stop Apache on EC2-2:

sudo systemctl stop httpd

After failed health checks, the target becomes Unhealthy and the ALB routes traffic only to EC2-1. Recover the service with:

sudo systemctl start httpd

Troubleshooting notes

Request timed out or 504 Gateway Timeout

The ALB could receive internet requests but could not reach the instances. Add an inbound HTTP rule on port 80 to the EC2 security group, with the ALB security group as its source.

Apache default page

The file was accidentally named intex.html. The correct path is /var/www/html/index.html.

What I learned

Application Load Balancer, target groups, health checks, security groups, high availability, traffic distribution and automatic failover.

Cleanup

Avoid unnecessary costs. Delete the Application Load Balancer, target group, lab EC2 instances and security groups that are no longer required.
