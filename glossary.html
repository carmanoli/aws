Glossary

Short definitions of terms used in the labs.

A

ALB: Application Load Balancer, a layer 7 load balancer for HTTP and HTTPS traffic.

AMI: Amazon Machine Image, a template used to launch an EC2 instance.

ASG: Auto Scaling Group, which maintains and adjusts a fleet of EC2 instances.

Availability Zone: One or more isolated data centres within an AWS Region.

C–E

CloudWatch: AWS monitoring service for metrics, alarms and logs.

Connection draining: Allowing existing connections to finish before removing a target.

Desired capacity: The number of instances an ASG currently attempts to maintain.

EC2: Amazon Elastic Compute Cloud, a service providing virtual servers.

H–U

Health check: A recurring test that determines whether a resource can serve traffic.

Instance warmup: Initialization time before a new instance influences scaling metrics.

Launch template: A reusable EC2 launch configuration.

Scale in / scale out: Remove capacity / add capacity.

Security group: A stateful virtual firewall.

Target group: A group of resources that receives traffic from a load balancer.

User data: A startup script supplied when an EC2 instance is launched.
