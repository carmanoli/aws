Lab – EC2 + GitHub using a Launch Template

Objective

In this lab, you will learn how to deploy a static website automatically on an Amazon EC2 instance using a Launch Template and EC2 User Data.

When a new instance is launched, it will automatically:

  • Install Apache HTTP Server
  • Install Git
  • Clone a public GitHub repository
  • Publish the website to /var/www/html
  • Start the web server
  • Make the website available over HTTP

This Launch Template will be reused in the following labs:

  • Lab 12 – Application Load Balancer (ALB)
  • Lab 13 – Auto Scaling Group
  • Lab 14 – GitHub Actions
  • Lab 15 – Jenkins CI/CD

Step 9 – EC2 User Data

Paste the following script into the User Data section.

#!/bin/bash

# Update the operating system
dnf update -y

# Install Apache
dnf install -y httpd

# Install Git
dnf install -y git

# Enable and start Apache
systemctl enable httpd
systemctl start httpd

# Clone the GitHub repository
cd /tmp
git clone https://github.com/carmanoli/aws.git

# Publish the website
cp -r /tmp/aws/* /var/www/html/

# Configure permissions
chown -R apache:apache /var/www/html
chmod -R 755 /var/www/html

Create the Launch Template.

Troubleshooting

Website does not load

Verify

  • EC2 instance is running
  • Security Group allows HTTP
  • Instance has a Public IP
  • Apache service is running
sudo systemctl status httpd

User Data did not execute correctly

Check cloud-init logs

sudo tail -100 /var/log/cloud-init-output.log

Cannot install packages

If you see errors similar to

Curl error (28)
Timeout was reached

verify the Security Group outbound rules.

Required outbound rule

Type Destination
All Traffic 0.0.0.0/0

Without outbound Internet access, the instance cannot:

  • Install Apache
  • Install Git
  • Clone the GitHub repository

User Data only runs once

User Data executes only during the first boot.

If the initial execution fails, the recommended solution is:

  1. Fix the problem.
  2. Terminate the EC2 instance.
  3. Launch a new instance using the same Launch Template.

This approach follows Infrastructure as Code principles and avoids manual server configuration.

Professional Best Practices

  • Use Launch Templates instead of manually creating EC2 instances.
  • Automate server configuration using User Data.
  • Keep infrastructure reproducible.
  • Store application source code in GitHub.
  • Recreate infrastructure instead of manually fixing servers.
  • Reuse the same Launch Template across multiple environments.
  • Restrict inbound traffic whenever possible.
  • Always allow outbound Internet access if packages or repositories must be downloaded during instance initialization.
