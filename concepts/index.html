AWS Concepts

Longer explanations of the ideas used throughout the labs.

Application Load Balancer

An Application Load Balancer operates at OSI layer 7. It evaluates HTTP and HTTPS requests and forwards them to healthy targets through listener rules and target groups.

Target groups

A target group is a logical collection of destinations, such as EC2 instances. The load balancer forwards requests to registered healthy targets using the protocol and port configured for the group.

Health checks

Health checks periodically test whether a target can serve traffic. An ALB stops routing new requests to an unhealthy target. An Auto Scaling Group can also use ELB health information to replace an unhealthy instance.

Security groups

Security groups are stateful virtual firewalls. A useful pattern is to allow inbound HTTP on the ALB from the internet and allow inbound HTTP on the EC2 instances only from the ALB security group.

Launch templates

A launch template defines how EC2 instances are created, including the AMI, instance type, security groups, storage, IAM instance profile and user data.

Auto Scaling Groups

An Auto Scaling Group maintains a desired number of EC2 instances between a configured minimum and maximum. It can distribute instances across Availability Zones, replace unhealthy instances and change capacity in response to metrics.

Target tracking

A target tracking policy adjusts capacity to keep a selected metric, such as average CPU utilization, close to a target value. New instances need a warm-up period before their metrics provide a useful view of capacity.

Scale out and scale in

Scale out adds capacity. Scale in removes capacity. During scale in, connection draining allows an ALB target to finish existing requests before it is deregistered.
