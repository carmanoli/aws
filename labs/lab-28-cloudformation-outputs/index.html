Lab 28 – AWS CloudFormation Outputs

Objective

The purpose of this lab was to extend the existing CloudFormation stack by introducing the Outputs section. Outputs provide an easy way to expose useful information about a stack after deployment and prepare it for integration with other CloudFormation stacks.

Existing Infrastructure

  • CloudFormation Stack
  • EC2 Auto Scaling Group
  • Launch Template
  • Application Load Balancer
  • Target Group
  • Security Groups
  • Systems Manager Session Manager
  • CloudWatch
  • SNS
  • Lambda
  • EventBridge
  • CloudTrail
  • Parameter Store

No additional AWS resources were created during this lab. Only the CloudFormation template was enhanced.

Understanding CloudFormation Sections

Section Purpose
Parameters Receive values supplied by the user during stack creation or update.
Resources Create and manage AWS infrastructure.
Outputs Publish useful information after the stack has been deployed.

The logical flow is:

Parameters
      │
      ▼
Resources
      │
      ▼
Outputs

Outputs Added

The following Outputs were added to the existing stack.

Outputs:

  StackName:
    Description: CloudFormation Stack Name
    Value: !Ref AWS::StackName

  AutoScalingGroupName:
    Description: Auto Scaling Group Name
    Value: !Ref LabWebAutoScalingGroup
    Export:
      Name: lab-asg-name

  AutoScalingGroupARN:
    Description: Auto Scaling Group ARN
    Value: !GetAtt LabWebAutoScalingGroup.AutoScalingGroupARN
    Export:
      Name: lab-asg-arn

  LaunchTemplateName:
    Description: Launch Template Name
    Value: launch-web-template

  TargetGroupARN:
    Description: Target Group ARN
    Value: arn:aws:elasticloadbalancing:eu-north-1:xxxxxxxxxxxx:targetgroup/lab-web-tg/xxxxxxxx

Intrinsic Functions Used

!Ref

The !Ref intrinsic function returns the logical value of a CloudFormation resource.

Example: 
Value: !Ref LabWebAutoScalingGroup

!GetAtt

The !GetAtt intrinsic function retrieves a specific attribute from a CloudFormation resource.

Example: 
Value: !GetAtt LabWebAutoScalingGroup.AutoScalingGroupARN

Exporting Outputs

CloudFormation allows Outputs to be exported so that other stacks can reuse them.

Example: 
AutoScalingGroupName:
  Value: !Ref LabWebAutoScalingGroup
  Export:
    Name: lab-asg-name

This makes the value available to any other CloudFormation stack inside the same AWS Region.

Preparing for Cross-Stack References

Although no second stack was created during this lab, exporting Outputs prepares the infrastructure for CloudFormation Cross-Stack References.

Architecture: 
CloudFormation Stack A
│
├── Resources
├── Outputs
└── Exports
        │
        ▼
CloudFormation Stack B
        │
        ▼
Fn::ImportValue
This enables infrastructure to be divided into independent reusable stacks.

Benefits

  • Improves infrastructure documentation.
  • Publishes useful deployment information.
  • Enables resource sharing between CloudFormation stacks.
  • Supports modular infrastructure design.
  • Prepares for Nested Stacks and enterprise Infrastructure as Code practices.

Conclusion

In this lab the existing CloudFormation stack was enhanced without creating additional infrastructure. Outputs were introduced to expose useful stack information, and selected values were exported for future reuse. These concepts are fundamental for building modular CloudFormation architectures and provide the foundation for Cross-Stack References, which will be implemented in the next lab.
