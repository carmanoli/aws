Lab 27 – CloudFormation Parameters

Objective

Convert an existing CloudFormation template into a reusable template by introducing Parameters and updating the infrastructure through Update Stack.

Environment

  • Existing stack: lab26-asg-stack
  • Auto Scaling Group: lab-cfn-web-asg
  • Launch Template: launch-web-template

Parameters

Parameters:

  DesiredCapacity:
    Type: Number
    Default: 1

  MinSize:
    Type: Number
    Default: 1

  MaxSize:
    Type: Number
    Default: 2

Template Changes

MinSize: !Ref MinSize
MaxSize: !Ref MaxSize
DesiredCapacity: !Ref DesiredCapacity

Validation

  • Updated the existing stack.
  • Changed DesiredCapacity from 1 to 2 without modifying the template.
  • Verified automatic scale-out.
  • Changed DesiredCapacity back to 1.
  • Verified controlled scale-in.

Concepts Learned

  • CloudFormation Parameters
  • Default values
  • Intrinsic function !Ref
  • Update Stack
  • Reusable Infrastructure as Code
  • Parameter-driven deployments

Conclusion

The template is now reusable because deployment-specific values are supplied as parameters instead of being hardcoded. This reflects common enterprise CloudFormation practices.
