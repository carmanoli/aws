Lab 38 - CloudFormation Custom VPC

This laboratory introduces the creation of a completely custom Amazon VPC using AWS CloudFormation. Unlike previous labs that relied on the AWS Default VPC, this lab builds an isolated networking environment following AWS networking best practices.

Objectives

  • Create a Custom VPC using CloudFormation.
  • Create two public subnets.
  • Deploy the subnets across two Availability Zones.
  • Create and attach an Internet Gateway.
  • Create a Public Route Table.
  • Create the default Internet route.
  • Associate both subnets with the Route Table.
  • Validate the deployment using a Change Set.
  • Inspect the resulting network architecture.

Implementation Steps

Step 1 - Create the CloudFormation Template

A new CloudFormation template was created containing all networking components required for a basic public VPC.

Step 2 - Validate with a Change Set

Instead of immediately deploying the infrastructure, a Change Set was created. This allowed verification of every resource that would be added before execution.

Step 3 - Execute the Stack

After validation, the Change Set was executed successfully and the stack reached the CREATE_COMPLETE state.

Step 4 - Validate the VPC

  • CIDR Block: 10.0.0.0/16
  • DNS Resolution: Enabled
  • DNS Hostnames: Enabled

Step 5 - Validate the Public Subnets

Subnet CIDR Purpose
lab-public-subnet-a 10.0.1.0/24 Availability Zone A
lab-public-subnet-b 10.0.2.0/24 Availability Zone B

Both subnets automatically assign public IPv4 addresses to launched EC2 instances.

Step 6 - Validate Routing

The Route Table contains two routes:

Destination Target
10.0.0.0/16 local
0.0.0.0/0 Internet Gateway

This default route makes the associated subnets public.

AWS Best Practices

  • Always create production VPCs using Infrastructure as Code.
  • Separate networking from application resources.
  • Deploy resources across multiple Availability Zones.
  • Use meaningful tags.
  • Validate infrastructure changes using Change Sets.
  • Reuse Outputs in future CloudFormation stacks.
  • Keep networking modular for future expansion.

Enterprise Perspective

The network built in this laboratory represents the first stage of a production-ready AWS environment. Future laboratories will progressively extend this architecture by introducing private subnets, NAT Gateways, VPC Endpoints, Network ACLs, Flow Logs, and finally migrating the existing Auto Scaling Group and Application Load Balancer from the Default VPC into this custom network.

Conclusion

In this laboratory, a complete custom Amazon VPC was deployed using AWS CloudFormation. The deployment followed Infrastructure as Code principles, was validated using a Change Set, and implemented a highly available networking foundation with two public subnets. This environment will serve as the basis for all future networking laboratories, ultimately replacing the AWS Default VPC with a production-grade architecture.
