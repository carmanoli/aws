Lab 29 - AWS CloudFormation Cross-Stack References

Objective

Learn how multiple CloudFormation stacks can share information by using CloudFormation Exports together with Fn::ImportValue.

This lab demonstrates how independent stacks can communicate while remaining loosely coupled, a common design pattern used in enterprise AWS environments.

  • Auto Scaling Group exported through CloudFormation Outputs.
  • Exports available:

    • lab-asg-arn

New Concepts

Consumer Stack

A second CloudFormation stack was created:

lab29-cross-stack-consumer

Instead of creating infrastructure, this stack imported values exported by the main infrastructure stack.

Template Highlights

Fn::ImportValue: lab-asg-arn

These intrinsic functions imported values directly from another CloudFormation stack.

CloudFormation Dependency Model

After importing exported values, CloudFormation automatically establishes a dependency between both stacks.

While the consumer stack exists:

  • Exports cannot be removed.
  • Export names cannot be changed.
  • The exporting stack is protected from breaking dependent stacks.

This automatic protection prevents accidental infrastructure failures.

Enterprise Architecture

Cross-stack references are commonly used to separate infrastructure into independent logical layers.

Stack Typical Resources
Networking VPC, Subnets, Route Tables, Internet Gateway, NAT Gateway
Security IAM Roles, Security Groups, KMS Keys
Compute Launch Templates, Auto Scaling Groups, EC2 Instances
Application Load Balancers, Target Groups
Monitoring CloudWatch, SNS, Dashboards, Alarms

Each stack exports only the values that other stacks need, reducing coupling and simplifying maintenance.

Advantages

  • Infrastructure reuse.
  • Independent deployments.
  • Clear separation of responsibilities.
  • Reduced template complexity.
  • Easier maintenance.
  • Improved scalability.
  • Safer infrastructure updates.
