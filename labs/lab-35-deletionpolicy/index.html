Lab 35 - AWS CloudFormation DeletionPolicy

This lab explores the DeletionPolicy attribute in AWS CloudFormation. DeletionPolicy defines what happens to a resource when the CloudFormation stack is deleted or when the resource is removed from the template.

Lab Objectives

  • Understand the DeletionPolicy attribute.
  • Study Delete, Retain and Snapshot behaviors.
  • Observe CloudFormation resource deletion.
  • Identify which AWS resources support Snapshot.
  • Compare DeletionPolicy with UpdateReplacePolicy.
  • Use temporary resources to avoid affecting the production lab infrastructure.
  • Keep the AWS account clean after completing the lab.

Prerequisites

  • AWS Account
  • AWS CloudFormation
  • Amazon S3
  • Amazon EC2
  • Existing permanent CloudFormation infrastructure from previous labs

DeletionPolicy Overview

DeletionPolicy controls the lifecycle of resources when CloudFormation deletes them.

DeletionPolicy Behavior
Delete Delete the resource.
Retain Keep the resource after the stack is deleted.
Snapshot Create a snapshot before deleting the resource.

Part 1 – Delete Policy

Create the following temporary stack:

AWSTemplateFormatVersion: '2010-09-09'

Resources:

  Lab35DeleteBucket:
    Type: AWS::S3::Bucket
    DeletionPolicy: Delete

    Properties:
      BucketName: !Sub "lab35-deletionpolicy-delete-${AWS::AccountId}-${AWS::Region}"

Expected Result

  • Stack created successfully.
  • S3 bucket created.

Delete the stack.

Observed Result

  • CloudFormation deleted the stack.
  • The S3 bucket was automatically deleted.
DeletionPolicy Delete removes both the CloudFormation resource and the physical AWS resource.

Part 2 – Retain Policy

AWSTemplateFormatVersion: '2010-09-09'

Resources:

  Lab35RetainBucket:
    Type: AWS::S3::Bucket
    DeletionPolicy: Retain

    Properties:
      BucketName: !Sub "lab35-deletionpolicy-retain-${AWS::AccountId}-${AWS::Region}"

Expected Result

  • Stack created.
  • S3 bucket created.

Delete the stack.

Observed Result

  • Stack deleted.
  • S3 bucket remained.
  • Bucket no longer managed by CloudFormation.
Retain protects the physical resource from accidental deletion.

Part 3 – Snapshot Policy

AWSTemplateFormatVersion: '2010-09-09'

Resources:

  Lab35EBSVolume:
    Type: AWS::EC2::Volume

    DeletionPolicy: Snapshot

    Properties:
      AvailabilityZone: eu-north-1a
      Size: 1
      VolumeType: gp3

Expected Result

  • EBS volume created.
  • Volume remained unattached.

Delete the stack.

Observed Result

  • The EBS volume was deleted.
  • A snapshot was automatically created.
Snapshot provides backup protection before CloudFormation removes the resource.

Resources Supporting Snapshot

Common supported resources include:

  • Amazon EBS Volumes
  • Amazon RDS DB Instances
  • Amazon RDS DB Clusters
  • Other storage-related resources supporting snapshots

Resources such as S3 buckets, Security Groups, IAM Roles, Load Balancers and Auto Scaling Groups do not support Snapshot.

DeletionPolicy vs UpdateReplacePolicy

DeletionPolicy UpdateReplacePolicy
Applies when a stack is deleted. Applies when CloudFormation replaces a resource during an update.
Protects deleted resources. Protects replaced resources.
Useful for preserving data after stack deletion. Useful for preserving data during infrastructure updates.

Enterprise Best Practices

Typical production configurations:

DeletionPolicy: Retain
UpdateReplacePolicy: Retain

or

DeletionPolicy: Snapshot
UpdateReplacePolicy: Snapshot

These configurations help protect critical business data during both stack deletion and infrastructure replacement.

AWS Console Observation

During this lab it was observed that uploading templates through the AWS CloudFormation console automatically stores copies inside an AWS-managed bucket named: 

cf-templates-xxxxxxxx-region

These uploaded templates are temporary artifacts created by the console and are not part of the managed infrastructure. Deleting this bucket does not affect existing CloudFormation stacks. The bucket is recreated automatically the next time a template is uploaded.

Conclusion

In this lab we explored one of the most important resource lifecycle controls available in AWS CloudFormation. Understanding when to use Delete, Retain and Snapshot is essential for designing safe Infrastructure as Code deployments. These concepts are widely used in enterprise environments to protect business-critical data and are frequently assessed in the AWS Certified Solutions Architect – Associate (SAA-C03) certification.
