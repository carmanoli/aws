Lab 22 – AWS Systems Manager Session Manager

Objective

Access an Amazon EC2 instance securely without using SSH, SSH keys or opening TCP port 22 by using AWS Systems Manager Session Manager.

Learning Objectives

  • Understand how Session Manager works.
  • Connect to an EC2 instance directly from the AWS Console.
  • Understand the relationship between SSM Agent, IAM Roles and Systems Manager.
  • Replace traditional SSH administration with Session Manager.
  • Reduce the attack surface by removing inbound SSH access.

Existing Infrastructure

  • EC2 instance: lab-web-server
  • Launch Template: lab-web-template
  • Security Group: lab-ec2-web-sg
  • Application Load Balancer
  • AWS Systems Manager
  • IAM Role: lab-ec2-ssm-role
  • CloudTrail

Prerequisites

  • EC2 instance running.
  • Amazon SSM Agent installed.
  • IAM Role attached to the instance.
  • AmazonSSMManagedInstanceCore policy attached.
  • Network connectivity to Systems Manager endpoints.

Verify Managed Node

  1. Open AWS Systems Manager.
  2. Open Fleet Manager.
  3. Verify that the instance appears as a Managed Node.

Open a Session

  1. Open EC2 Console.
  2. Select the instance.
  3. Choose Connect.
  4. Select the Session Manager tab.
  5. Click Connect.

A browser terminal should open without requiring an SSH client or key pair.

Verify the Session

whoami
hostname
pwd

Expected user:

ssm-user

Understand the ssm-user Account

id
sudo whoami

The Session Manager creates or uses the dedicated ssm-user account. Administrative commands are executed through sudo, following Linux best practices instead of logging in directly as root.

Verify the SSM Agent

Check whether the agent is running:

sudo systemctl status amazon-ssm-agent

Or simply verify its state:

sudo systemctl is-active amazon-ssm-agent

Install the SSM Agent (if required)

Amazon Linux 2 and Amazon Linux 2023 normally include the agent by default.

sudo dnf install amazon-ssm-agent
sudo systemctl enable amazon-ssm-agent
sudo systemctl start amazon-ssm-agent

Verify the IAM Role

Open the EC2 instance details and verify that the attached IAM Role is:

lab-ec2-ssm-role

The role must include the managed policy:

AmazonSSMManagedInstanceCore

Compare SSH and Session Manager

SSH Session Manager
TCP Port 22 HTTPS (443)
SSH Key Pair IAM Authentication
SSH Client Required AWS Console or AWS CLI
Internet Exposure No Management Ports Exposed
ec2-user ssm-user

Remove SSH Access

Edit the EC2 Security Group and remove the inbound TCP port 22 rule.

Validation

SSH test:

ssh -i "EC2-FIRST-KEY.pem" ec2-user@PUBLIC-IP

Expected result:

Connection timed out

Session Manager test:

echo "Session Manager is still working"

sudo systemctl status httpd --no-pager

The commands should execute successfully even though SSH is no longer available.

Key Takeaways

  • Session Manager provides secure remote administration without SSH.
  • No inbound management ports are required.
  • Authentication is controlled through IAM.
  • Administrative access can be audited through CloudTrail.
  • The attack surface is significantly reduced.

Troubleshooting

Problem Possible Cause
Instance not listed as Managed Node Missing IAM Role or SSM Agent
Cannot start a session SSM Agent stopped or no network connectivity
SSH still works Port 22 still allowed in the Security Group
Session Manager unavailable Verify IAM Role, SSM Agent and network connectivity

Conclusion

This lab demonstrated how AWS Systems Manager Session Manager can completely replace traditional SSH administration while improving security, simplifying key management and reducing the exposed attack surface of Amazon EC2 instances.
