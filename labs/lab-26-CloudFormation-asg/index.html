Lab 26 – CloudFormation + Auto Scaling Group

Objective

Create an Auto Scaling Group (ASG) using AWS CloudFormation, reuse an existing Launch Template, associate the ASG with an existing Target Group, and manage capacity using CloudFormation Update Stack.

Prerequisites

  • Launch Template: launch-web-template (Version 2)
  • Application Load Balancer: lab-web-alb
  • Target Group: lab-web-tg
  • Security Groups already configured
  • Session Manager enabled

Architecture

CloudFormation
      │
      ▼
Auto Scaling Group
      │
      ▼
Launch Template
      │
      ▼
EC2 Instances
      │
      ▼
Target Group
      │
      ▼
Application Load Balancer

CloudFormation Template

AWSTemplateFormatVersion: '2010-09-09'
Description: Lab 26 - CloudFormation Auto Scaling Group

Resources:
  LabWebAutoScalingGroup:
    Type: AWS::AutoScaling::AutoScalingGroup
    Properties:
      AutoScalingGroupName: lab-cfn-web-asg
      MinSize: '1'
      MaxSize: '2'
      DesiredCapacity: '1'
      VPCZoneIdentifier:
        - subnet-0d6b5a638e1b4b6f8
        - subnet-0e0cd186d22846c61
      LaunchTemplate:
        LaunchTemplateName: launch-web-template
        Version: '2'
      TargetGroupARNs:
        - arn:aws:elasticloadbalancing:eu-north-1:122445003699:targetgroup/lab-web-tg/50f8ce015d21e4c9
      HealthCheckType: ELB
      HealthCheckGracePeriod: 120
      Tags:
        - Key: Name
          Value: lab-cfn-web-instance
          PropagateAtLaunch: true

Deployment

  1. Open CloudFormation.
  2. Create a new stack.
  3. Upload the YAML template.
  4. Name the stack lab26-asg-stack.
  5. Wait for CREATE_COMPLETE.

Validation

  • Verify the ASG lab-cfn-web-asg exists.
  • Confirm one EC2 instance is launched automatically.
  • Verify the instance is automatically registered in lab-web-tg.
  • Wait until the target becomes Healthy.
  • Access the ALB DNS name to verify the application.

Update Stack

Edit the template:

DesiredCapacity: '2'

Run Update Stack. CloudFormation updates the ASG, which launches a second EC2 instance automatically.

After validation, edit the template again:

DesiredCapacity: '1'

Run another Update Stack. The ASG performs a controlled scale-in and terminates one instance after deregistration from the Target Group.

Key Concepts Learned

  • Infrastructure as Code with CloudFormation
  • Auto Scaling Group creation
  • Launch Template reuse
  • Automatic Target Group registration
  • Application Load Balancer integration
  • CloudFormation Update Stack
  • Declarative infrastructure changes
  • Scale-out and scale-in using Infrastructure as Code

Conclusion

This lab demonstrated how CloudFormation manages an Auto Scaling Group throughout its lifecycle. Unlike a standalone EC2 instance, an ASG automatically launches, registers, deregisters and terminates instances while maintaining the desired capacity defined in the CloudFormation template.
