Lab 24 – AWS CloudFormation (Introduction)

Objective

Learn the fundamentals of Infrastructure as Code (IaC) using AWS CloudFormation by creating, updating and deleting a CloudFormation Stack.

Prerequisites

  • Existing AWS account.
  • Default VPC available.
  • One existing subnet.
  • CloudFormation service enabled.

Concepts Covered

  • Infrastructure as Code (IaC)
  • CloudFormation Templates
  • Stacks
  • Parameters
  • Resources
  • Outputs
  • Stack Updates
  • Change Sets
  • Drift Detection
  • Stack Deletion

Template Used

AWSTemplateFormatVersion: '2010-09-09'

Description: Lab 24 - CloudFormation Introduction

Parameters:

  VpcId:
    Type: AWS::EC2::VPC::Id

  SubnetId:
    Type: AWS::EC2::Subnet::Id

  LatestAmiId:
    Type: AWS::SSM::Parameter::Value<AWS::EC2::Image::Id>
    Default: /aws/service/ami-amazon-linux-latest/al2023-ami-kernel-default-x86_64

Resources:

  LabSecurityGroup:
    Type: AWS::EC2::SecurityGroup
    Properties:
      GroupDescription: Security Group created by CloudFormation
      GroupName: lab-cfn-web-sg
      VpcId: !Ref VpcId

      SecurityGroupIngress:
        - IpProtocol: tcp
          FromPort: 80
          ToPort: 80
          CidrIp: 0.0.0.0/0

  LabEC2Instance:
    Type: AWS::EC2::Instance
    Properties:
      ImageId: !Ref LatestAmiId
      InstanceType: t3.micro
      SubnetId: !Ref SubnetId
      SecurityGroupIds:
        - !Ref LabSecurityGroup
      Tags:
        - Key: Name
          Value: lab-cfn-ec2

Outputs:

  SecurityGroupId:
    Value: !Ref LabSecurityGroup

  InstanceId:
    Value: !Ref LabEC2Instance

Procedure

  1. Create a new CloudFormation Stack.
  2. Upload the YAML template.
  3. Provide VPC and Subnet parameters.
  4. Create the Stack.
  5. Review Resources, Parameters and Outputs.
  6. Modify the template by adding an HTTP ingress rule.
  7. Create and review a Change Set.
  8. Execute the Change Set.
  9. Run Drift Detection and verify the stack is IN_SYNC.
  10. Update the Stack to add an EC2 instance.
  11. Verify the new EC2 instance was created.
  12. Delete the Stack and verify that both the EC2 instance and Security Group are automatically removed.

Troubleshooting Performed

  • Launch Template update failed because an incorrect Launch Template name was used.
  • The laboratory was simplified to keep the focus on CloudFormation fundamentals.
  • Rollback behavior and CloudFormation Events were used to identify the failure.

Key Learning Points

  • CloudFormation manages infrastructure declaratively.
  • Parameters make templates reusable.
  • Outputs expose useful resource identifiers.
  • Change Sets allow reviewing changes before execution.
  • Drift Detection identifies manual configuration changes.
  • Deleting a Stack automatically removes managed resources in the correct dependency order.

Result

The laboratory successfully demonstrated the complete CloudFormation lifecycle: Create → Update → Drift Detection → Delete.
