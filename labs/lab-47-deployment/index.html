Lab 47 – Private Website Deployment from Amazon S3

Objective

Replace the GitHub-based deployment with a production-style deployment using a private Amazon S3 bucket while keeping EC2 instances inside private subnets and without requiring a NAT Gateway.

Architecture

Private S3 Bucket
        │
Gateway VPC Endpoint
        │
EC2 Auto Scaling Group
        │
Apache
        │
Application Load Balancer

Starting Environment

  • Custom VPC with public and private subnets
  • Gateway VPC Endpoint for Amazon S3
  • Application Load Balancer
  • Launch Template and Auto Scaling Group
  • Systems Manager Session Manager
  • Private EC2 instances

Implementation

1. IAM Least Privilege

The EC2 IAM Role was updated to allow only:

  • s3:ListBucket
  • s3:GetObject

2. Upload Website

The website was uploaded to:

s3://lab-s3-endpoint-<account-id>-<region>/website/

3. Launch Template

User Data was modified to:

  • Install Apache
  • Install AWS CLI
  • Synchronize website from S3 using aws s3 sync
  • Configure Apache permissions
  • Start Apache

4. Bucket Policy

A Bucket Policy was added allowing only the EC2 IAM Role to read the website.

Validation

  • CloudFormation Update completed successfully.
  • New Launch Template version created.
  • Auto Scaling instance replaced.
  • Target Group reported Healthy.
  • Website served correctly through the ALB.
  • No Internet or NAT Gateway required.

Enterprise Relevance

  • Infrastructure separated from application artifacts.
  • Least Privilege IAM.
  • Private deployment through VPC Endpoint.
  • CloudFormation remains the single source of truth.
  • Foundation for CodePipeline, CodeBuild and CodeDeploy.

AWS SAA-C03 Skills Covered

  • Amazon S3
  • IAM Roles
  • Bucket Policies
  • Gateway VPC Endpoints
  • Launch Templates
  • Auto Scaling Groups
  • Application Load Balancer
  • CloudFormation

Conclusion

The application deployment process now follows a production-oriented design. Infrastructure and application artifacts are separated, EC2 instances use temporary IAM credentials, and deployments operate entirely within private networking using Amazon S3 and a Gateway VPC Endpoint.
