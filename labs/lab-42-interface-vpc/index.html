Lab 42 – Interface VPC Endpoints (AWS PrivateLink)

Objective

The objective of this laboratory was to understand how AWS PrivateLink provides private connectivity to AWS services by creating Interface VPC Endpoints inside private subnets.

Unlike Gateway Endpoints, Interface Endpoints create Elastic Network Interfaces (ENIs) inside the selected subnets and allow services to be reached using private IP addresses.

Laboratory Objectives

  • Understand AWS PrivateLink.
  • Compare Gateway Endpoints and Interface Endpoints.
  • Create an Interface Endpoint using CloudFormation.
  • Understand why ENIs are created.
  • Deploy the endpoint across multiple Availability Zones.
  • Associate a Security Group.
  • Understand Private DNS.
  • Learn enterprise design considerations.
  • Remove temporary resources to avoid unnecessary charges.

Architecture

 +------------------------------+ | AWS Systems Manager | +--------------+---------------+ ^ | AWS PrivateLink | +---------------+---------------+ | | ENI (10.0.11.230) ENI (10.0.12.117) | | Private Subnet A Private Subnet B | | Custom VPC (10.0.0.0/16)

Why Interface Endpoints?

Many AWS services cannot use Gateway Endpoints. Instead, they expose private connectivity through AWS PrivateLink.

When an Interface Endpoint is created AWS automatically provisions one Elastic Network Interface inside every selected subnet.

Applications communicate with those private IP addresses instead of using the public Internet.

Elastic Network Interfaces (ENIs)

The Interface Endpoint automatically created two ENIs:

Availability Zone Subnet Private IP
eu-north-1a lab-private-subnet-a 10.0.11.230
eu-north-1b lab-private-subnet-b 10.0.12.117

These ENIs became the private network interfaces used by Systems Manager.

Private DNS

Private DNS was enabled during endpoint creation.

Instead of resolving:

 ssm.eu-north-1.amazonaws.com

to a public IP address, DNS resolution returned the private IP addresses of the Interface Endpoint.

As a result, traffic remained entirely inside the AWS network.

Gateway Endpoint vs Interface Endpoint

Pricing Free Hourly + Data Processing

Enterprise Considerations

Large organizations commonly deploy Interface Endpoints for:

  • AWS Systems Manager
  • Secrets Manager
  • CloudWatch Logs
  • EC2 API
  • Amazon ECR
  • AWS KMS
  • AWS STS

This architecture allows workloads running in private subnets to communicate with AWS services without requiring Internet access or a NAT Gateway.

Cost Optimization

Interface Endpoints incur hourly charges and data processing costs. To avoid unnecessary expenses, the endpoint and its dedicated Security Group were removed from the CloudFormation template after completing the laboratory.

Conclusion

This laboratory introduced one of the most important networking services in AWS. Interface Endpoints enable secure, private communication with AWS services without exposing workloads to the Internet. Combined with Gateway Endpoints, they form the foundation of highly secure enterprise VPC architectures and are widely used in production environments where minimizing Internet exposure and implementing least-privilege networking are key design goals.
